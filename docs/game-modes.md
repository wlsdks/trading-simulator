## 0. 한 줄 결론

**개인/대회 2모드 분리는 채택한다(조건부 GO).** 이것은 신규 기능이 아니라, 이미 문서 곳곳에 흩어져 있는 정책(ACT-08 모드 격리·ACT-11 리필·ACT-07 다계좌·GRF-13 그룹대회·LEG-04 상금·MON-01 개설 게이팅)을 **유저가 이해하는 하나의 멘탈모델**로 묶고, 코드에는 하나의 계좌 축으로 배선하는 정리 작업이다. 8개 좌석 전원과 3개 적대 검수가 방향에 동의했다.

단, **오너의 표현 두 곳은 기존 문서와 정면충돌하므로 반드시 교정한다:**
1. **"개인 = 사실상 무한 리필" → "무한 리셋(재도전) + 상한 있는 리필"** 로 재정의. 무한 현금 주입은 ACT-11·앱스토어 시뮬레이션 도박 조항에 저촉되고, 게임적으로 손실 무게감을 0으로 만들어 재미를 죽인다.
2. **대회 상품(prize)은 v1(최소 1년) 전면 비현금·무참가비.** 참가비 + 현금성 상금 + 랜덤워크 = illegal lottery 3요소. 참가비/상금 금전 파라미터는 UI·API·DB 어디에도 만들지 않는다(구조적 차단).

그리고 적대 검수가 발굴한 **미해결 핵심 3개(격리만 있고 다리가 없음 / reEntry 시드 치트 / 방장 시드 정찰)를 선제 교정**하지 않으면 2모드는 "개인=이탈싱크, 대회=콜드스타트 붕괴"로 귀결된다. 이 문서는 그 교정을 포함한 정본이다.

## 1. 2모드 전략 — 판단과 근거

### 1.1 판단: 조건부 채택(GO with corrections)

| 관점 | 결론 | 핵심 근거 |
|---|---|---|
| 게임 디자인 | 조건부 Yes | 2모드 = 무한 샌드박스(캐주얼) × 유한 스테이크(랭크/토너먼트). 개인=폭·리텐션, 대회=깊이. 경쟁 아니라 퍼널 |
| 경제·밸런스 | 조건부 Yes | 무한 경제(희소성 0) vs 희소 경제(고정 시드). 개인 성과는 랭크 신호로 무의미 → 랭크 미노출 필수 |
| 대회 시스템 | 조건부 Yes | 실제로는 3계층(개인 / 공식 대회 / 방장 대회). '리필 토글'은 reEntry로 분리 봉합 |
| 디스커버리 UX | 조건부 Yes | 6번째 탭 금지, [랭킹] 하위 세그먼트. 콜드스타트를 관전+공식 시드 대회로 방어 |
| 수익화 | 조건부 Yes | 개인=광고·IAP 볼륨, 대회=구독 전환·K-factor. "무한은 시간이 지탱, 즉시성을 판다" |
| 법무·컴플라이언스 | 조건부 Yes | 대회의 유한 자본·리필 통제가 '우연' 의존을 낮춰 오히려 방어에 유리. 금전 파라미터 원천 차단 조건 |
| 트레이딩·회계 | 조건부 Yes | 신규 계좌 개념 아님 = ACT-07 kind의 유저 표면 매핑. 분모 일반화만 필요 |
| 프로덕트 전략 | 조건부 Yes | 흩어진 정책의 통합 프레임. 단 개인이 유저풀을 먼저 만들고 대회 탭은 공식으로 선채움 |

### 1.2 왜 좋은가 (재미의 척추)

- **개인 = "넓고 낮은" 재미**(누구나·언제나·부담 없이), **대회 = "좁고 높은" 재미**(준비된 유저·이벤트·강한 스테이크). 오너 가설("재미의 대부분은 대회에서")은 *깊이* 관점에선 맞지만, *폭·리텐션* 관점에선 개인모드가 캐리한다. 둘은 경쟁이 아니라 **퍼널**이다.
- 벤치마크 정합: 하스스톤(캐주얼/랭크/투기장), 포커(캐시게임/프리즈아웃 토너먼트), e스포츠 사설방(방장·초대코드). 특히 포커의 교훈 = **리바이(rebuy) 허용을 대회 파라미터로 명시**해야 공정. 우리 대회는 ACT-11로 리필 금지가 기본이므로 '노 리바이 프리즈아웃'이 디폴트.

### 1.3 교정 A — '무한 리필' → '무한 리셋'

재미의 최소 조건은 *결과가 걸려 있음(stakes)*이다. 현금을 무한 주입하면 매수 결정에 긴장이 사라져 클리커 게임이 된다. 게임적으로 옳은 자유는 **판 안에서는 손실이 아프되, 판을 언제든 새로 깔 수 있는 것** = 리셋의 자유이지 리필의 자유가 아니다. 포커 플레이머니 앱이 정확히 이 모델(칩 녹으면 무한 주입이 아니라 '새 바이인')이다.

- 리필은 ACT-11 그대로 유지: 일반 5%/일($5,000)·회당 $1,000~2,000·4h 쿨다운·1일 3회, 파산회생 3%.
- 재시작 욕구는 리셋(ACT-02: 24h 쿨다운·자발적, 파산 시 월 3회 상한 + 강제 복기 게이트)으로 흡수.
- 마케팅·앱스토어 카피에서 "무한 자금 / 무한 재시작 / 무한 리필" 문구 전면 금지(LEG-01, Apple 4.3 프레임 회피).

### 1.4 교정 B — 개인모드는 격하된 튜토리얼이 아니라 완결된 캐주얼 게임

'개인=연습장' 프레이밍은 유저가 "진짜는 대회구나" 하며 개인모드에 진행감을 못 느끼고, 대회에 진입 못 하는 대다수(친구 없음·저활동 시간대·초보)를 이탈시킨다. 개인모드는 **독립 진행 루프(레벨 언락·배지 컬렉션·시나리오·로컬 스트릭)를 온전히 소유한 '캐주얼 본게임'**이어야 하고, 대회는 그 위에 얹히는 '랭크/스테이크 레이어'다.

### 1.5 [핵심 교정 — 적대 검수 발굴] 격리만 있고 '다리'가 없다

현재 8개 설계는 전부 **격리(isolation)**를 강제한다: 0% 이월·원장 분리(ACT-07)·개인 랭크 미노출·개인 클라 시드 vs 대회 서버 시드. 그 결과 두 경제를 잇는 유일한 다리가 XP/배지 정체성뿐이라, **개인→대회로 넘어갈 기계적 당김(hook)이 0**이다. 이대로면 개인모드는 "넘어갈 이유가 없는 이탈싱크" 또는 "잘못된 손실 본능(손실=리셋 한 탭 세탁)을 심는 오훈련 튜토리얼"이 된다. **오너의 '개인=온보딩' 직관은 현재 스펙상 성립하지 않는다.**

이 문서는 다음 4개 다리를 정본에 포함해 해결한다(§2.4, §2.5, §6.5, §8):
- **(a) 기계적 CTA 다리**: 개인 시나리오/미션 클리어가 '이번 주 리그 도전' 원탭 CTA를 발화, 첫 미션 완료 후 주간 공식 리그 **opt-out 자동 등록**(결정 없이 디폴트 경로가 대회에 착지).
- **(b) 리셋에 지워지지 않는 성과-정직 서피스**: 리셋해도 누적되는 **개인 평생 실력지표**(TWR/리스크조정 복합스코어 히스토리). 리셋은 '새 시즌'을 열되 실력 트랙은 계속 쌓여 진짜 연습 피드백 + 대회 진입 명분('네 실력지표가 리그 입장선 도달')을 제공.
- **(c) 고스트/벤치마크 대회**: 30인 라이브 리그가 아니라 **로스터 독립형** 하우스·테마 시즌 대회(결정론·비동기, 시스템 고스트봇 상대)로 솔로 유저도 1일차에 완결 스테이크 경험.
- **(d) 프레스티지 소스 분리**: 대회 전용 프레스티지(트로피·타이틀·명예의전당)는 개인 활동으로 절대 획득 불가, 개인 XP는 개인 코스메틱/레벨만 구동.

## 2. 개인 모드(Personal) 설계

### 2.1 정의와 계좌 축

개인 모드는 별도 진입 화면이 아니라 **홈·마켓·자산의 기본 상태**다. 대회 미참가 유저 = 개인 모드. 현재 컨텍스트는 **상시 모드 배지**(개인 / 대회명·D-3)로 화면 상단에 노출(ACT-08 배지 규칙 준수).

계좌 속성은 **2축 직교**로 분리한다(가장 중요한 정합 이슈):

```
account.contextType = PERSONAL | CONTEST   // 경제 축(신규) — 리필·랭크 자격 결정
account.frictionMode = EASY | REAL         // 마찰 축(기존 ACT-08) — 체결가 결정
```

- 매핑: 개인 = `ACT-07 kind:MAIN`(EASY 기본, REAL 토글 실험 허용), 대회 = `kind:COMPETITION` 서브계좌.
- 두 축을 하나로 뭉개면 "리필 켜진 리얼 대회계좌" 같은 모순·격리버그가 난다. **2필드 분리가 하드 전제.**

### 2.2 경제 — 무한 리셋 + 상한 리필

| 항목 | 값 | 근거 |
|---|---|---|
| 시작자본 | $100k 고정(ACT-01) | 표준 분모 |
| 리셋 | 언제나 무료, 24h 쿨다운(자발), 파산 시 월 3회 상한 + 강제 복기 게이트 | ACT-02 |
| 리필 | 일반 5%/일·회당 $1~2k·4h·3회, 파산회생 3% | ACT-11 |
| 리필 경로 | `credit(type=AD_REWARD\|PROMO)` → `externalFlows` 기록 → **분모 차감**(ACC-06/ACT-03) | 리필해도 순위 못 부풀림 |
| 광고 회생 | 강제 금지, 항상 무료 리셋 병행 | ACT-02/ACT-11 |
| 코인/젬 → 매매원금 | 하드 금지 | ACT-11, MON-04, PRD §8.3 |
| 순위 | 글로벌 랭크/보상 리더보드 **미노출**, 코스메틱 보상만 | 무한 경제라 절대손익·수익률이 순위 신호로 무의미 |
| 시세 | 클라 결정론(PRD IC-1), 서버 스트림 미연동 → 인프라 비용 0에 수렴 | GRF-11 |

**핵심: 개인의 '점수'는 화폐가 아니라 재미지표다.** 무한 리셋 아래에서 '자기 최고 수익률'은 운 나올 때까지 리셋하는 reset-gacha로 파밍되므로, 점수는 **탐험한 종목 수·수집·스트릭·미션 클리어·평생 실력지표(§2.5)**로 재정의한다.

### 2.3 비용 충당(수익화)

- 광고(리워드/전면) + IAP(구독·코스메틱·계좌 슬롯). 코어 루프·매수 시그널 페이월 금지(MON-01).
- **"무한은 시간이 지탱하고, 즉시성을 판다"**: 리필 캡·쿨다운의 *기다림*을 (a) 리워드 광고 시청(MON-04 상한 내), (b) Plus 구독(쿨다운 스킵·소프트캡 상향)으로 즉시 해소. 무료 유저에겐 무한이되 느리고, 유료/광고 유저에겐 즉시 — 규제 정합적이면서 광고 임프레션·구독 가치를 동시 견인. 단 앱스토어 도박 조항 대비 **시간 마찰은 존치**(리필을 즉시 무제한으로 파는 순간 도박 프레임).

### 2.4 진행 루프(독립·완결)

- 온보딩(PRD 30초 첫 매수·컨페티) → 개인 샌드박스 착지.
- 레벨/배지로 상품군 언락 게이팅(P1 현물→레버리지→P2 옵션…)해 진행감 제공. **단 고위험(레버리지>임계·파생·옵션매도)은 GRF-16 고위험 리그로 성과 격리, 강한 면책·미성년 차단(GRF-07).**
- 첫 미션 완료 후 챌린지·랭킹 탭 점진 공개(PRD IA) → **주간 공식 리그 opt-out 자동 등록**(§1.5-a, 대회의 첫 접점, 친구 불필요).

### 2.5 [신규] 리셋에 지워지지 않는 성과-정직 서피스

- **개인 평생 실력지표**: 리셋해도 누적되는 리스크조정 복합스코어(TWR 기반) 히스토리 1개 신설. 리셋 = '새 시즌' 시작이되 실력 트랙은 계속 축적.
- **효과**: (1) 활동/수집과 무관한 '진짜 연습 피드백' 제공(현재 스트릭은 로그인+아무 액션 1개라 트레이딩 실력과 무관), (2) '네 실력지표가 리그 입장선 도달' 같은 대회 진입 명분, (3) reset-gacha 파밍 무력화(누적 지표는 운뽑기로 못 씻음).
- **정직성 워터마크**: 클라 결정론으로 생성된 개인 성과는 공유카드·프로필에서 **'비검증(연습)' 워터마크 강제**. '검증 실력지표'는 서버 권위 대회 성과에만 부여(GRF-11 확장) — 클라 시드 예지로 조작된 '완벽 이력'이 대회 진입 전 신뢰(herding·사칭)를 훼손하는 것을 차단.

### 2.6 프레스티지 소스 분리(대회 희소성 보호)

- 개인 XP → 개인 코스메틱/레벨만 구동. **대회 전용 프레스티지(트로피·타이틀·명예의전당)는 개인 활동으로 절대 획득 불가.**
- 크로스 이월은 중립적 '레벨/아바타'만. XP를 **소스별 태깅**(personal / contest / verified-official)하고, 상품 언락·연령·고위험 게이팅에는 **'검증된 진행'만 카운트** — 대회 담합→XP→개인모드 고위험 상품 조기 언락 우회로 차단(자산 0% 이월은 지켰어도 '정체성 이월'이 게이트를 뚫는 홀 봉쇄). 담합 무효화(GRF-14) 시 파생 XP·언락도 소급 회수.

## 3. 대회 모드(Contest) 설계

### 3.1 3계층 구조

'개인 vs 대회'는 계좌 kind에 매핑되지만, '대회'가 두 종류라 유저에게는 3계층으로 보인다.

| 계층 | 계좌(ACT-07) | 리필 | 보상순위 | 재원 |
|---|---|---|---|---|
| 개인 모드 | MAIN/PRACTICE, EASY 기본 | ACT-11 가드레일 내 | 글로벌 미연동(opt-out 자동등록 시 EASY 공식 리그만) | 광고·IAP |
| 대회 – 공식 | COMPETITION(시스템 생성) | **전면 금지** | 공식 티어·승강·명예의전당 글로벌 트랙 | 서버 결정론 시드 |
| 대회 – 방장 | COMPETITION(방장 생성) | **전면 금지** | 그룹 순위 격리(글로벌 미반영, GRF-13) | 방장 파라미터 |

**모든 주간 리그·하이롤러·그룹대회는 CONTEST 우산 아래로 통일한다.** 주간 리그 = "시스템이 방장인 상설 대회"(신선 시드·리필 봉쇄·시즌 계좌). 영속 개인 계좌는 순수 무한 리셋 샌드박스로 두고 랭크에서 완전 제외 → ACT-11의 "리필=비경쟁 맥락"이 "리필=PERSONAL"로 1:1 매핑되어 GRF-03 중립화 로직이 "CONTEST엔 외부유입 자체가 없다"는 불변식으로 단순화. (마이그레이션 비용 → grandfathering·90일 고지 필요, MON-01 clawback 반발 완화.)

### 3.2 방장 파라미터 스키마(생성 시 고정, 합류 후 변경 절대 불가 — GRF-13)

`Contest = { id, hostUserId, type, visibility, ruleset, capacity, schedule, joinPolicy, rewardSpec, status }`

`ruleset`(대회 생성 시 고정, 첫 합류/시작 시 LOCK):
- `startingCash` — 기본 $100k, **하드 클램프 $10k~$10M**(분모 붕괴·자릿수 어뷰징 방지, ACT-01 확장)
- `allowedInstruments` / `allowedMarkets` — 유니버스(KR/US/JP · 현물/ETF/옵션/선물/무기한/코인)
- `leverageCap` — 기본 1x, 1x~1000x. **>3x는 자동 '고위험 대회' 플래그**(GRF-07/16)
- `shortAllowed` — true 시 '원금초과 손실 가능' 고지 게이트
- `frictionMode` EASY|REAL — 참가자 전원 강제 동일(개인 성향 무시, ACT-08 엣지)
- `partialFill`, `rankMetric` {복합스코어(기본) | TWR | 규칙준수 | 학습}
- `refillPolicy` — **기본 OFF**(희소 경제). ON이면 externalFlow 기록 후 순위 전액 중립화 또는 '생존용 비순위' 라벨. 리필 ON 대회는 공식 글로벌 티어 미반영(공정성 격리)
- `bankruptcyPolicy` ∈ {ELIMINATE(기본) | RESET | NONE}(§6.2)
- `seed` — **서버 결정론 시드, `seed = H(competitionId, serverSecret)`로 대회별 유니크 파생**(MKT-12/GRF-11, §3.5 정찰 방어). engineVersion/seedVersion pin

`schedule`: `startTs`, `endTs`(고정 창) 또는 `rolling`
`capacity`: Free ≤3 / Plus ≤10 / Pro ≤50 / B2B 시트(MON-01)
`visibility`: public | private(링크만) | invite(승인제)
`joinPolicy`: `joinWindow` {preStartOnly(기본) | lateJoinAllowed}, `reEntry` {off(기본) | 재분기 시드·비순위}(§3.5)
`rewardSpec`: **앱 발행 비현금 카탈로그에서만 선택. 참가비 필드 없음(구조적 부재, §5).**

> **'리필 on/off' 요구의 봉합**: ACT-11이 대회 계좌 현금 리필을 전면 금지하므로, 방장 토글은 '리필'이 아니라 `reEntry`(파산 후 재도전 허용 여부)다. 계좌 안 현금 주입은 대회에서 항상 OFF·고정.

**방장은 참가자다.** 순위·체결·시세 특권 없음. 방장 특권은 생성·초대·(DRAFT)파라미터·조기종료뿐. 임의 강퇴 불가(신고→모더레이션만, GRF-14). 방장 잠적 시 고정 파라미터로 자동 진행·정산(방장 의존 제거).

### 3.3 격리 서브계좌 모델

- 참가자마다 `COMPETITION` 계좌: 독립 cash/holdings/orders/ledger, `competitionId` FK. 대회 종료 후 아카이브.
- **동일 시드 = 전 참가자 동일 가격경로**. 슬리피지/스프레드는 본인 주문 notional만의 순수함수(개인 독립 유동성 풀, 공유 오더북 없음) → 결정론·감사 성립(PRD §7).
- 계좌 간 자산 이전 전면 금지(ACT-07) → 개인→대회 자금세탁·동일종목 차익 원천 차단.
- **대회당 userId 1계좌 강제**(GRF-10), 다중계정 탐지 시 정지(ACT-06).
- 리필 화이트리스트 제외(ACT-03/ACT-11) — 분모 순수성 유지.

### 3.4 라이프사이클 상태기계

`DRAFT → RECRUITING → LIVE → SETTLING → ARCHIVED / HALL_OF_FAME`

- **DRAFT**: 방장만 파라미터 수정. 하이롤러/레버리지 조합 시 심사노트·면책 라벨 자동 부착. **방장 자기 계좌 드라이런/프리뷰 불가**(§3.5).
- **RECRUITING**: 초대링크/QR 발급(만료·인원상한·재사용 제한·승인제). 첫 합류 또는 시작 시점에 ruleset LOCK. 합류 시 COMPETITION 서브계좌 생성.
- **LIVE**: 서버 결정론 시드 스트리밍, 실시간 리더보드. 파라미터 변경·리필·임의 강퇴 불가. 오프라인 stale 주문 서버 거부(ORD-03/GRF-04).
- **SETTLING**: endTs 도달 → 전 서브계좌 스냅샷 동결, 열린 주문 일괄 취소·구속 해제, **서버 권위 재계산으로 최종순위 감사 확정**(GRF-01/02 위험조정 복합스코어), 비현금 보상 배분, realizedPL 원장 freeze(ACC-16), 이의제기 창(GRF-14).
- **ARCHIVED / HALL_OF_FAME**: 서브계좌 아카이브 스냅샷 보존(감사·리플레이), 잔액 현금가치 0 소멸(LEG-08). **그룹 대회 성과는 글로벌 티어 미반영**, 공식 시즌 대회만 명예의전당 글로벌 트랙 등재.

### 3.5 [P0 교정 — 적대 검수] 시드 무결성

**두 개의 치명적 홀을 선제 봉쇄한다(이 교정 없이 대회 보상 레이어를 켜면 안 됨):**

1. **reEntry 미래예지 치트**: reEntry 계좌가 같은 competitionId의 같은 시드를 다시 보면, 담합자/방장이 1차 시도로 가격 궤적 정찰→고의 파산→재입장 후 '알고 있는 미래'로 매매하는 완전 치트가 성립한다. GRF-11 '서버 은닉 시드'로도 못 막는다(같은 유저가 같은 시드를 두 번 경험하는 것 자체가 문제).
   - **교정**: 결정론·보상 대회에서 reEntry 기본 금지. 방장이 켜도 재입장 시 **시드 재분기** `seed' = H(seed, attemptIndex)`로 다른 시계열 부여, **재입장 계좌는 항상 비순위 + '연습' 라벨**. "같은 시드 재실행"으로 구현하는 순간 대회 무결성 붕괴.
2. **방장 시드 사전정찰(reconnaissance)**: 방장은 유일하게 '대회를 만들 수 있는' 참가자라, 동일 파라미터 비공개 대회를 반복 생성→관찰→취소하며 시드 궤적을 정찰한 뒤 본 대회를 여는 정보비대칭이 가능.
   - **교정**: `seed = H(competitionId, serverSecret)`로 대회마다 유니크 파생('같은 파라미터=같은 시드' 관계 단절), 방장 생성 후 시작 전 드라이런/프리뷰 불가, 보상·공개 대회는 방장 우승률을 GRF-15 안티메트릭으로 상시 감시.

### 3.6 순위 산정

- 기본 GRF-01 위험조정 복합스코어(MDD·분산·규칙준수 병기, GRF-08). 방장이 `rankMetric`을 절대수익으로 잡아도 **위험조정 병기 강제**; 절대수익 단일정렬은 하이롤러 격리 세그(GRF-07)에서만·강한 면책·미성년 차단.
- 타이브레이크 결정론. 지표 게이밍(막판 몰빵) 저항: 기간 일관성·규칙준수율 가중.
- 그룹 순위 ↔ 글로벌 티어 분리(GRF-13). 리그 세그 키 = `contextType × frictionMode × ruleset해시 × 숙련 × 자산군`(GRF-06/16). SOC-04 다축 필터는 **표시 필터일 뿐 보상 순위 재정의 금지.**

## 4. 디스커버리 탭 · 참여 · 관전 UX

### 4.1 정보구조 — 6번째 탭 금지

PRD 7.1은 5탭(홈·마켓·챌린지·랭킹·내자산)+점진공개가 확정 원칙. 대회 디스커버리는 신규 탭이 아니라 **[랭킹🏆] 탭 상단 세그먼트**:

```
[랭킹🏆]
 ├─ 리그      (GRF-05 — 기존)
 ├─ 대회 ★NEW (둘러보기 · 내 대회 · 만들기)
 └─ 리더보드   (SOC-04 다차원 — 기존)
```

- 점진공개 정합: '대회' 세그먼트는 첫 미션 완료·서버 활성 이후 노출. 게스트에는 지연가입 넛지(UC-B03).
- **개인 모드는 탭이 아니다**: 화면이 아니라 '어떤 계좌로 거래 중인가'로 표현 → 상시 모드 배지(개인 / 대회명·D-3).

### 4.2 둘러보기 — 레일 구성

가로 스크롤 카드 레일 5종(표시·탐색용, 공식 보상 순위와 무관 — SOC-04 라벨):

| 레일 | 소스 | 가드 |
|---|---|---|
| 🔥 인기 | 공개 대회 중 참가/관전/초대전환 상위 | 어뷰징 방어 후 집계(봇 유입 제외, GRF-10/11) |
| ✨ 신규 | 최근 개설·참가창 오픈 공개 대회 | 모더레이션 통과분만 |
| 👥 친구 | 팔로우/친구가 개설·참가한 공개+친구공개 | SOC-02 공개범위 존중 |
| 🏖 스폰서/공식 | 운영·브랜드 스폰서 대회 | 비현금만·'현금 아님' 라벨 강제(LEG-04) |
| ⏳ 마감임박 | 참가창/종료 임박 | 카운트다운 뱃지 |

**대회 카드(요약)**: 대회명·호스트 배지·모드 배지(개인규칙/EASY|REAL·레버리지 상한·공매도·리필 有無)·기간(D-3/진행중)·현재인원/정원·비현금 보상 아이콘·상태칩. **상금액 비노출**(LEG 관점).

- **공개 vs 비공개**: 디스커버리엔 공개+친구공개만. 비공개는 링크/QR 전용. 공개 대회 제목·설명·썸네일은 **SOC-07 UGC 파이프라인(필터→신고→모더레이션) 통과가 노출 전제**(하드 게이트). PII·리딩방 유인·외부 QR/링크 차단.
- **하이롤러/레버리지 대회는 디스커버리 기본 숨김·성인·옵트인**(GRF-07). 상단은 공식 리그·건전 챌린지 우선.

### 4.3 참여 플로우 — 조건 미리보기 → 원탭 참가

3경로 모두 **동일한 조건 미리보기 카드**로 수렴: (a) 디스커버리 카드, (b) 초대 링크/QR(deferred deep link, UC-B15), (c) 공유 카드 '참가하기'.

미리보기 카드 예시:
```
[대회명]                      [호스트 프로필]
 • 시작자본: $100,000
 • 기간: 8/1 09:00 ~ 8/8 15:30 (KST, 서버 UTC 기준)
 • 모드: REAL(수수료·슬리피지) · 레버리지 ≤3x · 공매도 허용
 • 허용 상품: US 현물 + ETF (옵션·선물 제외)
 • 리필: 금지(파산 시 관전 전환)   ← ACT-11
 • 순위기준: 복합 스코어(위험조정 병기)  ← GRF-01
 • 정원: 12 / 30 · 참가창 마감 D-1
 • 보상: 🏅배지 · 프로필 코스메틱 (현금 아님)  ← LEG-04
[관전하기]           [참가하기 →]
```
- **원탭 참가**: 가입·연령게이트·전면 면책 동의(대회 진입 직전 트리거, PRD 7.2, LEG-02/05) 통과 후 서브계좌 생성 + 서버 시드 스트림 합류. 미가입 게스트는 지연가입.
- **고위험 고지 세트**: 레버리지>임계·공매도·파생은 미리보기에 '원금 초과 손실 가능' 고지를 규칙과 세트로(GRF-16).
- **모드 강제 고지**: 대회 규칙이 REAL이면 유저 개인 성향 EASY여도 대회 서브계좌는 대회 규칙셋 강제(ACT-08 엣지).

### 4.4 관전(Spectator) — 콜드스타트·퍼널의 핵심

- **관전 범위**: 라이브 순위표(닉네임·복합스코어·수익률 라벨)·남은 시간·규칙 배지·참가자 수만. **타 참가자 실시간 포지션·주문은 비공개**(SOC-06 herding/카피 방지). 청산 이력은 사후 요약만·조롱 브로드캐스트 금지(SOC-05).
- **[P1 교정 — 적대 검수] 담합 피드백 루프 차단**: 진행 중 보상 대회의 실시간 리더보드는 **참가자에게만** 공개, **비참가 관전자에겐 지연·순위 대략화**. 관전 계정과 참가 계정에 linkage(GRF-10)가 걸리면 담합 가중신호로 처리. (담합조가 관전 계정으로 순위 델타를 실시간 모니터하며 참가 계정에 '지금 밀어줘/버텨' 타이밍을 주는 것을 막음.)
- 관전자는 매매 불가(서브계좌 미생성·읽기 캐시·비과금). '참가하기' CTA 상시 노출.
- **호스트 토글**: 관전 허용 on/off. 교실(B2B)·비공개는 기본 off 가능.
- 관전 카운트는 표시용이며 보상과 무관(봇 관전 인플레는 GRF-11로 정제).

### 4.5 라이브 대시보드 · 공유(K-factor)

- 상단: 대회명·규칙 배지·카운트다운·내 순위·1등과 격차. 서버 권위 스냅샷 주기 갱신(GRF-04/11, 기기시각 불신).
- 접근성: 2초 시세 갱신 aria-live=off, 순위 변동·체결·에러만 assertive announce(PRD 7.5).
- 대회 계좌는 리필 버튼 미노출(ACT-11). '이 순위는 대회 내 순위·글로벌 미반영' 라벨(GRF-13).
- **공유 카드**: 대회 성적·배지·티어 중심 + **모드 라벨 강제 표기**(EASY↔REAL 억지 비교 방지) + 하단 미세 면책 강제. 절대수익 자랑 절제(SOC-06). 초대 딥링크 결합('나 이기러 와' / '다음 시즌 열림').

### 4.6 콜드스타트 방어(반대 시나리오 = 텅 빈 탭)

탭이 죽는 이유는 언제나 **공급(방장 부족)**이다. 대비:
- 디스커버리 상단은 유저 대회 0개여도 **공식 대회가 항상 채운다**: 주간 공식 리그(GRF-05) + 월간 테마 시즌 대회('코인 위크', '방어적 분산 챌린지'를 하우스 상시 운영).
- **[P0 교정] 진짜 콜드스타트 필러는 로스터 독립형**: GRF-05 주간 리그는 풀<30명 시 '비경쟁 연습'으로 강등되어(저활동 시간대·소규모 지역) 콜드스타트가 무는 바로 그 순간 개인모드로 회귀한다. 따라서 **지수/시스템 고스트봇 필드 대비 채점되는 하우스·테마 시즌 대회(결정론·비동기)**를 상시 운영해 솔로 유저도 1일차에 실제 스테이크·리더보드가 있는 완결 대회를 경험. 고스트는 '시스템 상대'로 명시(가짜 인간 노출은 LEG/평판 리스크). 30인 라이브 리그는 세그먼트 유동성이 생긴 뒤로 유보.

## 5. 상품(Prize) 정책 — 안전한 구조 권고

### 5.1 결론

**v1(최소 1년): 앱이 직접 거는 현금·현금성(기프트카드·상품권·포인트)·참가비 전면 금지.** 참가비 + 현금성 상금 + 랜덤워크(우연) = illegal lottery 3요소. 특히 랜덤워크 시세는 '실력이 우연을 지배' 방어가 구조적으로 약하다(LEG-04). **앱이 걸 수 있는 것은 앱 발행 비현금 카탈로그뿐.**

### 5.2 허용 / 금지 / 미래 경로

- **허용(v1)**: 배지·프로필 코스메틱/플레어·리그 티어·명예의전당 기록·챔피언 타이틀. (양도·환급·현금교환 불가 약관 명시로 2차시장 현금화 차단, LEG-08.)
- **금지**: 현금·현금성 등가물, 유료 참가비, 프리미엄 전용 참가(=사실상 대가 → **무료 참가경로 반드시 병설**, 구독료=참가비 오인 차단), 관전자 베팅 유료화.
- **구조적 차단**: 대회 생성 UI·API·**DB 스키마 어디에도** 상금액/참가비/rake/pot 파라미터를 만들지 않는다. 필드가 존재하는 순간 방장이 오프플랫폼 도박을 운영하는 방조 표면이 열린다.
- **조건부 미래 경로(v2+, 법률자문 후 3요소 중 하나 제거 원칙)**: 스폰서 상금(대가 미성립) → 스위프스테이크 AMOE(대가 제거) → 실데이터(우연 제거). 각 단계 Official Rules·지오펜싱·세무(1099/W-8BEN)·주별등록·앱스토어 5.3 콘테스트 선언·operator-of-record 스폰서 명확화 게이트. **rake(참가비 일부 수취) 절대 금지**(대가+상금 동시 성립).

### 5.3 [P0 교정 — 적대 검수] '엔트리 토큰/명성 스테이크' 개념 폐기

수익화 설계가 제안한 '엔트리 토큰(reputation stake)'은 **기존 문서에 없는 신규 발명이며 이번 설계에서 유입된 최대 도박 리스크다.** 토큰을 '걸고(stake) 지면 소각·이기면 몰아줌(pot)'하는 순간 비현금·비구매여도 '대가+상금' 구조가 재현된다(미국 다수 주는 시간·노력·소각 가능한 인앱자산도 consideration으로 광의 해석, 리워드 광고=대가 그레이존까지 겹침).
- **교정**: '스테이크/소각/풀/양도'가 전혀 없는 **순수 자격 언락**(play로 열리는 참가권, 결과와 무관하게 차감 안 됨)으로만 재정의하거나 개념 폐기. **설계에서 '스테이크'라는 단어를 제거.** 엔트리성 재화는 IAP SKU가 물리적으로 붙을 수 없는 코드경로에 격리하고 ADR로 판매 금지 명문화(향후 '토큰 판매' 오픈 = 즉시 복권화).

### 5.4 [P0 교정] KR/JP는 'AMOE=대가제거'가 이식 불가

패널이 '참가비 0 스폰서 대회 = 3요소 미성립 청정'으로 낙관했으나, 이는 US illegal-lottery(대가 제거) 프레임에 갇힌 판단이다.
- 한국 경품류제공고시(표시광고법)·일본 景品表示法은 **참가비 0이어도 '경품' 제공 자체에 상한**을 건다. LEG-04는 이 상한을 '미래 현금상금 경로'에만 걸어두고 **v1 스폰서 상품 경로에는 누락** → **하드 게이트로 추가 필요**(JP: オープン懸賞 vs クローズド懸賞 판정 — 구독/IAP 퍼널 안이면 상한 적용, KR: 현상경품 상한).
- AMOE(무료 참가경로 병설→대가 제거)는 **US 전용 세이프하버**로만 문서에 한정 표기. US에서도 무료경로가 유료경로와 완전 동등(동일 당첨확률·동등 노출)할 때만 유효 — 구독자에게 더 많은 참가·상위노출을 주면 disparity로 대가 잔존. **KR/JP 대회의 유일 안전선은 '현금성 상금 자체 제거(prize prong 제거)'.**

### 5.5 방장 오프플랫폼 상품 — 방조 표면 최소화(완전 차단 불가 명문화)

방장이 대회 설명에 '1등 치킨 쏨' 같은 오프플랫폼 상품을 거는 것: 앱은 에스크로·지급·정산·광고에 일절 관여 안 함. 그러나 앱이 순위·판정 엔진을 제공하는 이상 KR에서 '판정 도구 제공 = 방조'로 걸릴 수 있고, ToS+텍스트 필터만으로는 코드화된 우회('1등 선물💝 DM')를 못 막는다.
- **교정**: (a) 대회 결과에 '우승자 지정 + 상금청구 어포던스'를 UI에 만들지 말 것, (b) 비공개 그룹 순위는 계속 비권위(글로벌 미반영) 유지, (c) ToS에 오프플랫폼 현금대회 명시 금지 + 신고 + 제재 + 모니터링, (d) **잔존 리스크가 '완전 차단 불가'임을 명문화**('억제됨'은 과신).

### 5.6 회계 가드 + 구독권 상금 주의

- **prize는 절대 `cash`나 `credit()`(externalFlows)로 지급 금지** — 분모 오염(ACC-06) + 도박 프레임 동시 발생. 원장과 무관한 게임화 자산(배지/XP/코스메틱)으로만 지급.
- **가상현금 팟/상금 금지(코드레벨 불변식)**: 참가자 가상현금을 모아 우승자에 몰아주는 '팟'은 가짜돈이어도 심사자에게 도박 서사. 가상현금은 오직 '시드'로만 존재, 환전·이전·팟 부재를 **서버 스키마에서 강제**.
- **구독권 상금 이중 함정**: (1) 시장가치 있어 US 과세소득('비현금=세무무관'은 오류), (2) 앱 자신이 가격 매긴 '가치 있는 상금'이라 prize prong 강화. → **시장가격 없는 순수 코스메틱·타이틀·프레임·명예의전당 우선.** 구독권 사용 시 연간 누적가치 집계·양도불가·1099 임계 추적을 Official Rules에 명시하고 '어떤 대가와도 결합 금지' 불변식 추가.

### 6.1 대회 유형

| B 공식 이벤트/시즌 챌린지 | 유한 | 테마 제한 | 1x | 복합스코어 | 하우스 상시(콜드스타트 필러), 비현금 시즌 트로피 |
| C 방장 커스텀 그룹 | 방장 지정 | 자유·고정 | 방장 | 방장(위험조정 병기 강제) | 초대(K-factor), 그룹 격리 |

**[P0 교정] 고분산 파라미터 조합 방어**: `(leverage>3x AND duration<임계 AND 소인원)`은 랜덤워크에서 사실상 룰렛 → 자동 하이롤러 격리(GRF-07): 보상·글로벌 미반영, 배지 등급 하향, 디스커버리 기본 숨김. 담합자의 dispersion farming(소규모 대회 난사→하나에서 우승 배지) 차단. startingCash 클램프만으론 조합 폭발을 못 막음.

### 6.2 파산 처리

- **개인**: 파산(equity < 시작자산 20%, 히스테리시스 20%진입/25%해제) → 자동 리셋 안 함, 유저 확정. 무료 리셋 항상 병설, 광고 회생 강제 금지. 월 3회 상한 + 강제 복기 게이트. XP·배지·스트릭 불초기화(ACT-02).
- **대회**: `bankruptcyPolicy` — **ELIMINATE(권장 기본)**: DNF/최하위 고정, 매매 잠금 → 관전 전환 / **RESET**: RESET_BASELINE 이벤트 기록 + 이력 노출, 시즌 스코프 재배치(순위 세탁 방지, ACC-16) / **NONE**: 잔여자본으로 계속. 리필 금지이므로 회생 없음. 레버리지 대회 음의 equity는 ACC-22 보험기금 흡수 + 0클램프(유저 지갑 절대 음수 불가), 초과분은 원장 밖 `insuranceFundAbsorbedCents`로 상각.
- 초보 보호: ELIMINATE '잔여자본 계속' 프리셋 + 짧은 기간·낮은 레버리지 초보 대회 프리셋 제공(한 번 파산=끝까지 구경만 완화).

### 6.3 중도참여(late join)

- 개인: 언제나 참여(로스터 없음).
- 대회: 기본 `joinWindow=preStartOnly`(고정창 공정성, 저경쟁 풀 헌팅 방지). 방장이 lateJoinAllowed 켜면 **현재 시각·잔여기간으로 시작 + 별도 late 코호트/정규화 지표로 랭킹, 과거 시드 리플레이 금지**(미래 가격 예지 치팅 차단, GRF-03 중립화 재사용). 마감 후 접근자는 관전만 + 유사 공개 대회 제안.

### 6.4 중도이탈 / 정지 / 삭제

- 탈퇴: 잔존자 기준 순위 유지, 이탈자 무순위/자동 최하위. SUSPENDED 계정은 진행 대회 즉시 실격·순위 제외(ACT-06). 삭제 요청 시 실격 후 타 참가자 순위 무결성 보존(ACC-06).

## 7. 기존 계좌 / 회계 / 리더보드 정합

### 7.1 분모 일반화(최우선 델타)

현행 ACC-06/ACT-13/ACT-01은 `simpleReturn = (equity − startingEquity − Σ외부유입) / startingEquity`, 분모를 **전 유저 $100k 고정**으로 못박음. 대회는 방장이 startingCash를 정하므로 일반화 필수:

> 분모 = 계좌의 `startingEquityCents`. **하나의 리더보드(글로벌 리그 또는 특정 대회) 안에서는 전원 동일 startingEquity를 강제**한다. 개인 = $100k 고정, 대회 = 방장 startingCash(참가자 전원 동일이라 비교가능성 유지).

이렇게 하면 MKT-12(startingCash 대회 파라미터)와 정합하고 '비교가능성=같은 분모'가 대회 스코프 안에서 성립.

### 7.2 원장·격리 불변식

1. **원장 분리**: 개인·각 대회 서브계좌 독립(ACT-07), 계좌 간 이전 전면 금지 → 자금세탁 원천 차단.
2. **0% 이월**: 대회는 항상 방장 startingCash에서 재출발. 개인 성과·리필·XP로 얻은 것 어떤 것도 대회 시드에 합산 금지.
3. **랭크 자격 격리**: PERSONAL 계좌는 GRF-05 리그·SOC-04 보상 뷰 진입 불가. 노출은 탐색/친구뷰(비경쟁) 라벨만.
4. **화폐 3분리**: 코인/젬→시드·매매원금 구매 불가, 가상현금→현실/프리미엄 환전 불가(GRF-03·PRD §8.3).
5. **크로스 정체성만 이월**: XP·레벨·배지(소스 태깅) 영속 정체성만 공통 누적(ACT-02). 대회 프레스티지는 개인으로 획득 불가(§2.6).

### 7.3 시세 결정론

- 대회 안: 대회별 유니크 시드 서버 권위 결정론(MKT-12), 전 참가자 동일 시계열, 시드 클라 은닉, 종료 후 시드 보관→감사. 유동성=개인별 독립 풀+주문규모 결정론 충격함수.
- 개인끼리·대회끼리는 독립 시세. 개인은 클라 결정론 허용, 순위·보상 걸린 대회만 서버 권위 필수.
- 계좌별 활성 시세 소스 분리 → 같은 종목이라도 값 다름 → 대회 컨텍스트 배지 강제.

### 7.4 종료·아카이브

- SETTLING 시 realizedPL 원장 freeze + 아카이브(ACC-16 재계산 감사), 잔액 현금가치 0 소멸(LEG-08). 대회 순위 글로벌 티어 미반영(GRF-13). 뒤늦은 치팅 발각 시 배지·티어 소급 회수·승급자 재정산.
- 계좌 스위처: 개인·대회 잔액 합산 표시 금지, 계좌별 분리 표기 + 모드 배지(ACT-13 엣지, '내 돈' 오해·이전 착시 차단). 아카이브 서브계좌·프리미엄 만료 초과 계좌는 읽기전용 잠금.

## 8. MVP 배치(시퀀싱)

> **P1 용어 주의:** 이 섹션의 MVP/P1/P1.5~P2는 대회 모드 시퀀싱이다. build-ready P1 실행 범위와 IN/OUT은 [p1-scope.md](./p1-scope.md)를 SSOT로 따른다.

PRD MVP 원칙 = "혼자서도 돌아온다"(로컬·솔로 우선, 서버 post-MVP). 대회 모드 전체가 서버 백엔드(결정론 시드·리그 집계·권위 시각·이상탐지)에 강하게 종속(ACC-07 다계좌 리팩터, GRF-11).

| 단계 | 내용 | 근거 |
|---|---|---|
| **MVP** | 개인 모드 완성(무한 리셋·상한 리필·모드 배지·독립 진행 루프·**평생 실력지표**) **+ 콜드스타트 방어형 대회 프리미티브 1개**(고스트/벤치마크 비동기, 서버 경량 또는 클라 결정론+고정 벤치마크) | 로컬 리텐션 검증 + **대회 후크 조기 계측** |
| **P1** | 대회 Stage A = 비공개 그룹 대회(초대제, 디스커버리 탭 불필요, K-factor 엔진) + 공식 주간 리그 | 서버 도입 후, 탭 채우기 불필요 |
| **P1.5~P2** | 대회 Stage B = 공개 대회 디스커버리 탭(공식 시드 선탑재) + 관전 | UGC 모더레이션·연령게이트 하드 게이트 선행 |
| **이후** | 스폰서 상금(법률 게이트 통과 시), 하이롤러 격리 대회 | LEG-04 단계 조건부 |

### 8.1 [핵심 교정 — 적대 검수] MVP 검증 리스크

프로덕트 좌석은 'MVP=개인 단독'을 제안했으나, **패널 전원이 재미·리텐션은 대회에 있다고 논증**했다. 개인 단독 D7이 약하면 진짜 재미(대회) 출시 전에 제품을 오판 폐기할 수 있다.
- **교정**: (a) 개인 단독 D7을 '약한 프록시'로 명시적 라벨(PRD가 이미 D7을 가설로 강등 — 유지), (b) **콜드스타트 방어형 대회 프리미티브 1개를 MVP로 당겨** 대회 후크 자체를 조기 계측 — 재미 명제 전체를 P1로 미루지 말 것. 이 프리미티브는 로스터 독립형(고스트/벤치마크)이라 유저풀 없이도 성립.

### 8.2 [필수] 개인→대회 페어드 안티메트릭

'개인모드=이탈싱크'를 조기 감지할 지표가 현재 없다. tracking-plan에 추가:
- 개인→첫 대회 전환율, 첫 대회까지 중앙 소요시간
- 대회 참가자 중 순수 초대유입 vs 개인 출신 비중
- **페어드 경보**: 높은 개인 리텐션 + 낮은 대회 진입 ⇒ 개인모드가 종착 싱크라는 신호 → 리텐션 자축 금지, 퍼널 개입 트리거

### 8.3 4대 법적 가드레일 정합(재확인)

- 현금상금 X: LEG-04 비현금 강제(§5), 참가비 필드 구조적 부재
- 모의 면책: 대회 카드·공유카드에 시뮬 배지·모드 라벨 하드코딩
- 개인 매수추천 X: 대회는 유니버스·룰만, 개별 종목 시그널 금지(MON-01 R1)
- 실데이터 라이선스: 대회 시세 = 서버 시드 결정론 시뮬(MKT-12), 실시간 실데이터 아님
