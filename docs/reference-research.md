# 레퍼런스 조사 보고서 (Reference Research Report)

> **본 문서는 모의투자 시뮬레이션 앱 기획 문서이며 실제 투자자문이 아닙니다.** Trading Simulator는 가짜 자산으로 매매를 연습하는 페이퍼 트레이딩(모의투자) 앱으로, 실제 자금·실제 체결이 없으며 표시 시세는 시뮬레이션 데이터입니다.

**한줄요약:** 경쟁 앱·게임화·시세 API·수익화·규제 5개 축의 레퍼런스를 종합하면, 우리 앱은 "교육형 × 게임형의 교집합, 실전 온보딩형의 UX 완성도"에 포지셔닝하고, 게임화 보상을 '거래 빈도·수익률'이 아닌 '학습·리스크관리'로 재정의하며, 시세는 단계적으로 실데이터로 승격하되 재배포 라이선스를 확보하고, 수익화는 구독+리워드광고+코스메틱 IAP+B2B 하이브리드로, 현금 상금 대회와 유료 시그널은 규제상 초기 배제하는 것이 핵심 전략이다.

## 1. 조사 범위와 시장 지형

모의투자/페이퍼트레이딩 시장은 크게 세 갈래로 나뉜다.

| 구분 | 정의 | 대표 앱 | 핵심 무기 |
|---|---|---|---|

| **(2) 실전 온보딩형** | 실제 브로커/차트 플랫폼의 "실전 진입 전 연습장" | Webull Paper, thinkorswim paperMoney, TradingView Paper | 도구 깊이(지표·백테스트·스크립트) |

**우리의 포지셔닝:** "시장 무관 범용 시뮬레이터 + 게임화·소셜·학습·수익화"를 지향하므로 **(1)교육형과 (3)게임형의 교집합**에 서되, **(2)실전 온보딩형의 UX 완성도**를 벤치마크한다. 단, 우리는 브로커가 아니므로 (2)의 "무료 미끼→실계좌 전환" 수익모델은 복제 불가하다.

**업계 위생요인(hygiene factor) — 없으면 감점, 있어도 차별점 아님:**
- 시작 가상자금 **$100,000** (Investopedia·thinkorswim·TradingView 공통 — 우리도 이미 채택, 적절)
- 시장가/지정가/손절 주문
- 포트폴리오 손익 추적
- 공개/비공개 대회와 리더보드

## 3. 게임화·리텐션 패턴

### 3.1 핵심 원리: 레이어드 설계

핀테크·에듀 앱의 장기 리텐션은 단발성 보상이 아니라 **손실회피(streak) · 사회적 책임(리그·친구) · 성취 위계(배지/레벨)를 사용자 여정 단계별로 겹겹이 배치**하는 데서 나온다(Duolingo가 대표).

```
XP(공통 화폐)
  → 사용자가 직접 설정하는 데일리 목표
    → 스트릭(streak)
      → 승급/강등 있는 주간 리그(약 30명 풀)
        → 희귀 배지 · 친구 스트릭
```

여정 단계가 깊어질수록 새로운 후크가 켜져 '롱테일' 리텐션을 만든다.

### 3.2 검증된 벤치마크 수치 (Duolingo)

| 메커니즘 | 수치 | 시사점 |
|---|---|---|
| **스트릭 프리즈 완충** | 프리즈 사용자 평균 유지 **17.19일 vs 미사용 11.62일 (+48%)** | 완충 없는 스트릭은 한 번 끊기면 이탈 직결 → 프리즈 필수 |
| **온보딩 첫날 성취** | day-one achievement 완료자 리텐션 **33.42% vs 미완료 20.36%** | 첫 세션에 '보장된 작은 승리' 1개 필수 |
| **주간 리그** | 약 30명 소규모 풀 + 승급/강등 | 글로벌 단일 리더보드는 상위 고래만 보이고 대다수 포기(역효과) |

> **참고:** 데일리 퀘스트 DAU +25%, XP 리더보드 참여 +40%, 배지 완료율 +30% 등의 벤치마크가 보고되나 이는 출처의 자체 수치로 **검증 한계 있음(추정)**. 공통 원리는 **변동보상 · 진행감 · 손실회피 · 사회적 인정**의 조합이다.

### 3.3 게임화 메커니즘 카탈로그

| 메커니즘 | 후크 유형 | 모의투자 적용 시 주의 |
|---|---|---|
| 스트릭(streak) | 손실회피 | '매일 트레이딩'이 아니라 '매일 학습/체크인'에 걸 것 + 프리즈 완충 |
| 데일리 목표/퀘스트 | 진행감 | 소수(2~3개)로 제한해 강박·번아웃 방지 |
| XP·레벨 | 진행감 | 보상 대상을 학습·리스크관리로 |
| 배지 | 사회적 인정 | 첫날 배지는 '학습 시작' 성격으로 한정, 희귀 배지와 등급 분리 |
| 리그(승급/강등) | 사회적 책임 | 약 30명 풀, 위험조정 지표로 |
| 변동보상(보물상자) | 도파민 | **학습 완료 보상에만 한정**, 매매 결과와 절대 연동 금지 |
| 친구 스트릭 | 관계적 비용 | 개인 손실이 아닌 관계 비용으로 복귀 강제 |

### 3.4 규제 경고: 실거래 게임화는 이미 규제 표적

| 규제 주체 | 지적 내용 |
|---|---|
| **FCA (영국)** | 포인트/리더보드/배지/축하 애니메이션(컨페티)/잦은 푸시가 "투자자가 자기 이익에 반하는 행동을 하도록 유도" |
| **매사추세츠 규제당국 vs Robinhood** | 게임화가 투자를 "하찮게(trivialize) 만들고 잦고 위험한 거래를 조장" → 약 **$750만 합의 + 해당 기능 제거** |
| **OSC (온타리오)** | 소셜 피드·카피트레이딩이 군집행동(herding)을 키움 |

### 3.5 모의투자 특유의 최대 리스크: '나쁜 습관의 실전 이식'

우리 앱은 실제 금전 손실이 없지만, **게임 루프가 잦은 매매·수익률 추종을 학습시키면 유저가 실거래로 옮겨갈 때 도박적 행동을 그대로 가져간다.** 앱스토어 심사·규제 논리도 여기에 집중한다.

**→ 전략적 결론: 보상 함수를 재정의하라(최우선 원칙).** 스트릭·XP·배지·리그의 보상 대상을 '매매 횟수/수익률'이 아니라 **'학습 미션 완료·분산투자·손절 규칙 준수·복기 작성'**에 건다. 거래 빈도 지표는 낮아질 수 있으나, 이는 규제·앱스토어 방어선이자 차별화 포인트다. KPI를 '학습 세션 리텐션'으로 잡아야 자기모순이 없다.

## 4. 시세 데이터 API 비교표

### 4.1 핵심 결론

무료 티어의 실시간 시세는 거의 예외 없이 **개인/평가용으로 제한**되며, 앱 사용자에게 **재배포(redistribution)하는 순간 유료 상업 라이선스가 필요**하다. 모의투자 앱은 본질적으로 시세를 사용자 화면에 재배포하므로, **무료 API로 실서비스 재배포는 대부분 약관 위반**이다. 이것이 비용·법무의 진짜 병목이다.

## 6. 현실성 설계 · 규제/컴플라이언스

### 6.1 현실성(Realism) — 단계적 도입이 정석

현재 MVP(2초 랜덤워크, 시장가 즉시 체결, 수수료·슬리피지 없음)는 "게임"에 가깝다. 실전 이전(移轉) 학습 도구가 되려면 마찰 요소를 **단계적으로** 도입해야 한다.

| 단계 | 요소 | 난이도 |
|---|---|---|
| 1 | **주문유형**: 시장가·지정가·손절/익절/OCO | 낮음 |
| 2 | **체결 마찰**: 수수료(0.015~0.25%대 예시)·슬리피지·부분체결·호가 스프레드·체결지연 | 중 |
| 3 | **코퍼레이트 액션**: 배당·액면분할·병합 (자체 이벤트 스케줄러로 구현, 분할 시 보유수량·평단가 자동 조정) | 중 |
| 4 | **공매도·레버리지**: 강제청산(마진콜) 로직 + '손실이 원금 초과 가능' 고지 세트로만 출시 | 높음 |

**핵심 설계 원칙 — '난이도 모드' 분리:** 마찰을 초반부터 강하게 넣으면 신규 리텐션이 하락(추정)하므로 **'쉬운 모드(무마찰)'와 '리얼 모드(마찰 on)'를 토글로 분리**하고 리얼 모드를 상급 학습/대회용으로 배치. 단, **두 모드 성과를 같은 리더보드에서 비교하면 형평성 붕괴 → 모드별 리그 분리 필수.**

## 7. 우리 앱에 대한 종합 시사점

아래는 5개 조사 축을 관통하는 **실행 우선순위 종합**이다. 각 항목에 2차 효과(사이드이펙트)를 명시한다.

### 7.1 최우선 원칙 — 보상 함수 재정의 (게임화 × 규제의 교차점)

게임화 보상 대상을 **'거래 빈도·수익률'이 아니라 '학습·좋은 습관·리스크 관리'로** 재정의한다. 이것이 리텐션 엔진을 유지하면서 규제·앱스토어 방어선을 세우는 **우리 제품의 차별화이자 방어선**이다.
- 스트릭 → '매일 트레이딩'이 아니라 '매일 학습 미션 완료'
- 리더보드 → 절대수익률 금지, **위험조정 지표**(MDD 대비 성과·샤프 유사·분산 점수·학습 진도·규칙 준수율)
- 변동보상/컨페티 → **학습·리스크관리 축하에만**, 매매 결과와 절대 연동 금지
- **2차 효과:** 거래 빈도 KPI는 낮아지나 KPI를 '학습 세션 리텐션'으로 잡으면 자기모순 없음. FCA·Robinhood 규제 논리를 정면 회피.

### 7.2 온보딩 — '계정 없이 30초 내 첫 매수' + '보장된 작은 승리'

thinkorswim의 위압적 UI가 반면교사. 범용 초보 타겟에 대한 최대 차별점은 **'쉬움/즉시성'**. AsyncStorage 로컬 저장 구조상 게스트 시작이 쉬우므로 이 강점을 극대화하고, 첫 실행에서 **튜토리얼 매매 1건 + 즉시 배지(day-one achievement)** 지급(근거: 완료자 리텐션 33% vs 20%). 소셜/대회 참여 시점에만 **지연 가입** 요구.
- **2차 효과:** 성취 인플레이션 방지 위해 첫날 배지는 '학습 시작' 성격으로 한정, 희귀 배지와 등급 분리. 연령 게이트는 가입 퍼널 이탈 유발 → 최소 필드·명확한 사유 표기로 마찰 최소화.

### 7.3 학습·게임화 통합 (Wall Street Survivor 원형)

'첫 매수하기 / 분산 포트폴리오 만들기 / 손절 걸어보기' 같은 **과업형 미션 → 완료 시 배지 + 가상현금 크레딧**. 매매 화면 개념을 짧은 카드형 설명으로 즉시 연결(Investopedia식 맥락형 학습).
- **2차 효과:** 미션이 매매를 유도해 데일리 리텐션과 튜토리얼 완주율 동시 상승.

### 7.4 리더보드·리그 설계

**'리스크조정수익 + 리그/기간 분리'.** 약 30명 소규모 풀 + 승급/강등(이길 수 있는 경쟁), 주간/월간·초보/숙련 리그 분리. 도박 욕구는 별도 **'고위험 리그'**로 격리.
- **2차 효과:** 도박형 지표 억제 시 상위권 재미가 줄 수 있으나 고위험 리그로 욕구 격리. 절대수익 랭킹은 herding·모방매매 조장 및 '모의투자=일확천금' 프레임을 심어 실전 이식 위험 증대 → 배제.

### 7.5 소셜 — '그룹/길드 단위 비공개 대회'를 1순위 기능으로

Investopedia 비공개 게임 + Stock Wars 그룹 벤치마크. 친구 초대·그룹 리더보드가 **K-factor**를 만든다. 실시간 데이터·기술 영역에선 Best Brokers식 뉴스/RSS 통합, TradingView식 차트-매매 통합으로 세션 길이 증대.

### 7.6 수익화 — 브로커 전환 경로 없는 하이브리드

| 채널 | 내용 | 2차 효과 |
|---|---|---|
| 구독(주 수익) | 고급 통계·다계좌·광고제거·백테스트(Webull Replay 벤치마크). 월 $9.99·연 $59~79·체험 7일·Pro티어 신설 | '실시간 데이터 없음'이 약점으로 보일 수 있어 '연습 전용' 명확 포지셔닝 필요 |
| 리워드 광고 | 가상현금 리필·부스터·프리미엄 1일 체험 | **가상현금 무한 리필은 손실 무게감·희소성 훼손 → 일일 리필 상한·쿨다운 필수** |
| 코스메틱/편의 IAP | 테마·아바타·배지 + 슬롯·백테스트 횟수 | 게임 대비 IAP ARPU 낮으나 규제 마찰 감소 |
| B2B 라이선스 | 교사 대시보드·학급 대회·커리큘럼 연간 시트 | 별도 웹 콘솔·인보이스/세일즈 역량 필요 → 초기 리소스 분산 |

**'실전 투자 유도'·수익 보장 문구는 규제/앱스토어 리스크이므로 전면 배제.** 유료 대회는 최소 1년 보류.

### 7.7 시세 데이터 로드맵

현재 12종목·랜덤워크는 MVP로 충분하나 경쟁 앱(Best Brokers 6만 종목, TradingView 다자산)과 격차 큼. **단계적 승격**: 시뮬레이션 → Finnhub+Twelve Data 무료 베타 → 재배포 허용 유료 플랜. 크립토는 거래소 API 직접 연동($0). 한국은 초기 지연/일봉. **지금 `MarketDataProvider` 추상화와 백엔드 프록시를 설계**하지 않으면 재작업 비용 발생.
- **2차 효과:** 라이선스 비용이 고정비화 → 수익화(구독/광고)와 연동해 손익분기 유저 수 역산 필요.

### 7.8 현실성·규제 하한선

- 현실성은 **'쉬운/리얼 모드' 분리 + 모드별 리그**로 도입.
- **면책 한/영 병기**를 코드·카피·앱스토어 설명에 즉시 반영. Google Play Financial features declaration 제출.
- **연령 게이트·개인정보처리방침 선반영**(글로벌 배포 대비).
- **종목추천·개인화 시그널 유료화 배제**(자문업 트리거). AI 코치는 '과거 학습·일반 개념 설명' 프레이밍 유지.
- **현금 상금 대회 v1 배제**, 비현금 보상으로 시작.
- 공매도·레버리지는 **강제청산 로직 + 원금초과 손실 고지 세트로만**, 코퍼레이트 액션·주문유형 도입 이후 후순위.

## 8. 부록: 용어 · 출처 신뢰도 표기

### 8.1 용어

- **위생요인(hygiene factor):** 없으면 감점이지만 있어도 차별점이 되지 못하는 기본 기능.
- **재배포(redistribution):** 시세 데이터를 앱 사용자(제3자) 화면에 노출하는 행위. 대부분 무료 티어 약관이 금지.
- **learn-while-you-trade:** 학습 강좌와 실제 매매를 한 루프로 묶는 게임화 구조(Wall Street Survivor).
- **AMOE(Alternative Method of Entry):** 무료 참가 경로. 현금 상금 대회에서 '대가(consideration)' 요소를 제거해 복권 규제를 회피하는 장치.
- **Dominant Factor Test:** 실력이 우연을 지배함을 입증해 '실력 기반 콘테스트'로 인정받는 미국 판례 기준.
- **RLTV/ARPPU/eCPM:** 결제자 생애가치 / 결제자당 평균매출 / 유효 1,000노출당 광고수익.
- **K-factor:** 기존 유저 1명이 데려오는 신규 유저 수(바이럴 계수).

### 8.2 신뢰도 표기 원칙

본 문서에서 **(추정)**으로 표기한 항목(예: Webull $1M의 학습 전이 저하, 국내 순수게임의 리텐션 루프 약함, 마찰 도입의 신규 리텐션 하락, 게임화 벤치마크 %수치, 한국 실시간 라이선스 비용)은 업계 통념·출처 자체 수치에 기반한 것으로 **1차 검증이 필요**하다. 특히 재무 전제(전환율·eCPM·수수료)와 규제 조항(자본시장법·복권법·PIPA/GDPR/COPPA)은 **출시 전 법무·회계 실사로 확정**해야 한다.
